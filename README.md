
# Лабораторная работа №22. Интерактивные веб-приложения на JavaScript

## 1. Описание проекта
Данный проект представляет собой интерактивный веб-калькулятор, разработанный с использованием стандартных веб-технологий: **HTML5**, **CSS3** и **JavaScript (ES6+)**.
Проект демонстрирует навыки создания пользовательского интерфейса, обработки событий, работы с DOM и применения объектно-ориентированного программирования (ООП) в JavaScript. Код выложен в публичный репозиторий на GitHub для демонстрации навыков работы с системой контроля версий.

## 2. Функциональность

### Поддерживаемые операции
Калькулятор поддерживает базовые арифметические действия:
- Сложение (`+`)
- Вычитание (`-`)
- Умножение (`*`)
- Деление (`/`)
- Очистка текущего выражения (`C`)
- Вычисление результата (`=`)

### Ограничения
- **Ввод:** Ввод осуществляется только через кнопки интерфейса.
- **Дробные числа:** Поддерживается ввод чисел с плавающей точкой (`.`), но запрещен ввод более одной точки в одном числе.
- **Последовательность:** Запрещен ввод двух операторов подряд (например, `5++3`), за исключением смены знака.

### Обработка ошибок
- **Деление на ноль:** При попытке деления на ноль выводится сообщение "Ошибка".
- **Синтаксические ошибки:** Если введенное выражение не может быть вычислено (например, `5++` или `*5`), система перехватывает исключение и выводит сообщение "Ошибка", сбрасывая состояние калькулятора при следующем нажатии.

### Класс `Calculator`
Основная логика вынесена в отдельный класс `Calculator` (файл `calculator.js`), что соответствует принципам ООП:
- **Конструктор:** Инициализирует состояние (переменная `expression`, флаг ошибки `hasError`) и получает ссылки на DOM-элементы.
- **Метод `init()`:** Навешивает обработчики событий на кнопки интерфейса.
- **Метод `handleButtonClick(value)`:** Центральный диспетчер, определяющий действие в зависимости от нажатой кнопки.
- **Методы валидации:** `addToExpression`, `isValidExpression`, `hasDuplicateDot` — обеспечивают корректность ввода данных пользователем.
- **Методы вычисления:** `calculate`, `safeEvaluate` — выполняют математические операции с обработкой исключений.

### Схема взаимодействия компонентов
1. **HTML** создает структуру (дисплей и сетку кнопок).
2. **CSS** задает визуальный стиль (центровка через Flexbox, сетка кнопок через Grid).
3. **main.js** импортирует класс `Calculator` и создает его экземпляр при загрузке страницы.
4. **calculator.js** управляет состоянием: при клике на кнопку в HTML, JS обновляет внутреннюю переменную `expression` и обновляет значение в `input` на странице.

## 4. Логика работы

### Последовательность действий при нажатии кнопок
1. Пользователь нажимает кнопку на экране.
2. Срабатывает событие `click`, которое перехватывается методом `handleButtonClick`.
3. В зависимости от значения кнопки:
   - **Цифра/Оператор:** Проверяется валидность -> Добавляется в строку выражения -> Обновляется дисплей.
   - **C (Clear):** Сбрасывает строку выражения в пустую -> Обновляет дисплей.
   - **= (Equal):** Запускает процесс вычисления.

### Как работает валидация ввода?
Перед добавлением символа в выражение вызываются проверки:
- **Двойные операторы:** Проверяется последний символ строки. Если он оператор, новый оператор не добавляется.
- **Две точки:** Проверяется текущее число. Если точка уже есть, вторая не добавляется.
- **Начало выражения:** Запрещается начинать выражение с оператора (кроме минуса).

### Как происходит вычисление?
1. Метод `calculate()` берет текущую строку `expression`.
2. Вызывается вспомогательный метод `safeEvaluate()`, который использует встроенную функцию `eval()` внутри блока `try...catch`.
3. **Успех:** Результат вычисления записывается обратно в `expression` и выводится на экран.
4. **Ошибка:** Если `eval` выбрасывает исключение (например, синтаксическая ошибка), срабатывает блок `catch`, на экране появляется надпись "Ошибка", и флаг `hasError` устанавливается в `true`.
